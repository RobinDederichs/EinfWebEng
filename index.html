Aufgabe 1.1 Fachliche Argumentationen über Erfolgsprinzipien des WWW

1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?

Die Hypertext-Konferenz empfand die Ideen von TBL als rückständig zur aktuellen Entwicklung des Hypertexts, unter anderem Störte sie die nicht vorhandene referentielle Integrität der Hyperlinks

2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

TBL ermöglichte die Evolvierbarkeit. Ebenfalls gibt es im WWW keine zentrale Kontrollinstanz, was es zu einem der erfolgreichsten Medien der Welt machte, an dem jeder Nutzer frei partiziepieren kann.

3. Was wäre der Preis für die garantierte Verhinderung von “broken links”?

Eine zentrale Kontrollinstanz, weniger Freiheit und somit nicht die freie Gestaltung und Entwicklung des WWW

Aufgabe 1.2 HTTP

1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

Success, oder Success ohne Inhalt

2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

301 gibt mir den Hinweis zurück, dass die gesuchte Ressource dauerhaft auf eine neue Adresse verschoben wurde und liefert mir diese direkt mit

3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

400 bedeutet fehlerhafte Anfrage. Das bedeutet, dass ich meine HTTP-Anfrage kontrollieren muss

4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

403 bedeutet fehlende Zugangsberechtigung. Um dagegen etwas zu tun muss ich mir von dem Admin der Website erforderliche Rechte einholen

5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

Stackoverflow sagt: "It's not cacheable, period. Sorry."

Nach dem Standard RFC 3986 in BNF: URI ::= scheme ":" authority [ "/" path ] [ "?" query ] [ "#" fragment ]
